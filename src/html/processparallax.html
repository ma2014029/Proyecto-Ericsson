Process

¿Que es?

El proceso de desarrollo de software es una estructura aplicada al desarrollo de un producto de software Hay varios modelos a seguir para el establecimiento de un proceso para el desarrollo de software, cada uno de los cuales describe un enfoque diferente para diferentes actividades que tienen lugar durante el proceso. Hay varios procesos de desarrollo de software especificos que se ajustan a un modelo de ciclo de vida. Las actividades dependeran del modelo que se elija.
Modelos:
Cascada: El desarrollo en Cascada es el enfoque metodológico que ordena rigurosamente las etapas del proceso para el desarrollo de software de tal forma que cada inicio de una etapa debe esperar a la finalizacion de la anterior.

Espiral: El desarrollo en espiral es un modelo de ciclo de vida del software definido por Barry Boehm. Las actividades del modelo de espiral no estan fijadas sino que se eligen en funsion del analisis de riesgo comenzando por el bucle interior.

Modelo de prototipo: El modelo de prototipo pertenece a los modelos de desarrollo evolutivo. El prototipo debe ser construido en poco tiempo usando los programas adecuados y no se debe utilizar muchos recursos.

Desarrollo rapido de aplicaciones: El desarrollo rapido de aplicaciones o tambien conocida RAD es un proceso de desarrollo inicialmente por James Martin. Hoy en dia se suele utilizar para referirnos al desarrollo rapido de interfaces graficas de usuario

Modelo iterativo: Es un modelo derivado del ciclo de vida en cascada. Este modelo busca reducir el riesgo que surge entre las necesidades del usuario y el producto final por malos entendidos durante la etapa de recogida de requisitos.

Programacion Extrema: Es una metodológia ágil y flexible utilizada para la géstion de proyectos. Se centra en potenciar las relaciones interpersonales del equipo de desarrollo como clave del éxito mediante el trabajo en equipo.
