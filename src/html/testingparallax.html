Testing

¿Que es?

Las pruebas de software son las investigaciones empíricas y técnicas cuyo objetivo es proporcionar información objetiva e independiente sobre la calidad del producto a la parte interesada o stakeholder.
Es una actividad más en el proceso de control de calidad.
Las pruebas son básicamente un conjunto de actividades dentro del desarrollo de software.
Dependiendo del tipo de pruebas, estas actividades podrán ser implementadas en cualquier momento de dicho proceso de desarrollo.
Existen distintos modelos de desarrollo de software, así como modelos de pruebas.
A cada uno corresponde un nivel distinto de involucramiento en las actividades de desarrollo. Para esto existen los "testers", un tester investiga un producto de software con el objetivo de obtener información acerca de su calidad y del valor que representa quienes lo utilizan, el tester participa en todas las etapas del desarrollo de un software colaborando para asegurar la maxima calidad del producto.
Existen diferentes metodos de testing como por ejemplo:

Black Box: Es el metodo de prueba de software que se utiliza para probar el software sin conocer la estructura interna de su codigo.
White Box: Estas tecnicas se realizan cuando el tester accede al código fuente de la aplicacion y en consecuencia a los diferentes algoritmos y estructuras de datos utilizadas.
Gray Box: El concepto de caja gris en simple se basa en la relacion de caja negra basados en casos de prueba, realizados por personas que conocen el programa por dentro. La caja gris es un hibrido de prueba de Black Box y White Box.
GUI Testing: Actualmente la interfaz grafica de usuario suele ser una de las partes de un proyecto software que se prueba de forma menos completa y menos sistematica
